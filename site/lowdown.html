Introduction

Sow is a project scaffolding generator. Techincally it is a simple, plugable templating system based on eRuby, but it is designed with project scaffolding squarely in mind.

Installation

Install using RubyGems as you would expect:

  $ gem install sow

Manual site installation can be achived via Ruby Setup. Please see the README included in the tarball for more information.

Usage

To start a new Ruby project simply use:

  $ sow --ruby=myapp

You will see the following outout:

  Transferring ruby files to myapp/:

      create  bin
      create  data
      create  doc
      create  doc

With you project setup, be sure to go over the contents and fine-tune it. In particular, edit all the files that contain ‘__FIX__’ entries.

Hoe Users

Hoe users will notice that the sow command that comes with Hoe is overwritten by Sow. Sow supports Hoe projects, so there is nothing lost. Simply use:

  $ sow --hoe [directory]

Note that the Hoe scaffold does not support the Till out of the box. To get that you need to create a .meta direcroty in your project.

Designing Sow Plugins

Creating your own plugin is actually quite easy. You simply need to add a folder to a Ruby project's loadpath following the naming convention <name>.sow/. This folder can be in a dedicate project or part of another project. Typically your project's loadpath with be under lib/. Let's say you want to call you plugin mohgee, so you would have lib/mohgee.sow directory in your project. Aternatively you can make a separate folder for plugins, and add it's path to your project's loadpath (which is what I do). In that case you might have a folder with a pathname of plug/moghee.sow.

Within this folder you simply put whatever files you want to be apart of your scaffolding, using erb within them when they need to contain dynamically generated content.

Finally you need to add a script.sow.rb file to the folder. This script instructs Sow on how to conduct the file transfer and generation when the plugin is used. Essentailly, a Sow script is a small Ruby DSL script. There are only a few methods you need to know in order to create you own plugin.

Below is the script.sow.rb for the Ruby plugin. It covers the majority of available options.

  about "Scaffold a traditional ruby project."

  usage "--ruby[=<package-name>]"

  argument(:name, 'package name of new application/library') do |val|
    val = val || pathname
    abort "Name is required." unless val
    metadata.package = val
  end

  scaffold do
    copy('**/*', '.')
    copy('.meta/*', metadir)
  end
